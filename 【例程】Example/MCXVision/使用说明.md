# MCXVision 纯色沙包检测使用说明

## 1. 工程定位

本目录：

`D:\2026_SmartCar\MCX_Vision_Library-master\MCX_Vision_Library-master\【例程】Example\MCXVision`

当前版本已经从原来的模型识别流程切换为纯传统视觉方案，目标是稳定识别纯色沙包，并通过串口把目标边框坐标发给下位机做闭环控制。

当前版本特性：

- 摄像头采集分辨率为 `320x240`
- 使用 `KEY1` 对屏幕中心处的沙包颜色做一键标定
- 基于 `HSL` 阈值分割，不再依赖 AI 模型
- 处理流程为：粗网格筛选 + 连通域搜索 + ROI 精修
- 串口输出使用固定 `11` 字节协议
- 发送给下位机的坐标统一缩放到 `160x120` 逻辑坐标系
- 当前程序按接近每帧一次发送坐标，典型发送频率约 `30Hz`

## 3. 当前识别流程

每帧处理流程如下：

1. `SCC8660` 采集一帧 `RGB565` 图像
2. 在屏幕上显示实时画面，并绘制中心十字
3. 按下 `KEY1` 时，以屏幕中心附近小区域为样本自动生成目标颜色阈值
4. 以固定步长扫描整幅图像，构建粗网格颜色匹配图
5. 对候选网格做连通域搜索
6. 选择最优候选区域后，在 ROI 内做像素级边框精修
7. 输出检测框并通过 `UART5` 发送到下位机

## 4. 硬件连接

### 4.1 视觉板与下位机

视觉板与下位机串口连接关系：

## 6. 标定方法

### 6.1 标定步骤

1. 将纯色沙包放到屏幕中心十字附近
2. 尽量保持光照稳定
3. 按下 `KEY1`
4. 屏幕中心会显示红色反馈框
5. 调试串口会打印当前得到的 HSL 阈值

标定输出示例：

```text
[Calib] H:120-164 wrap:0 S:76-240 L:18-148
```

字段说明：

- `H`：色相范围
- `S`：饱和度范围
- `L`：亮度范围
- `wrap=1`：说明色相范围跨越了 HSL 环形边界

### 6.2 标定建议

- 标定区域里尽量只放沙包，不要混入背景
- 避免强反光和过曝
- 光照变化明显时重新标定
- 如果颜色和背景太接近，建议优先改善现场布置

## 8. 串口协议

### 8.1 串口参数

- 波特率：`115200`
- 数据位：`8`
- 停止位：`1`
- 校验位：`None`
- 发送频率：典型约 `30Hz`

### 8.2 数据帧格式

每帧固定 `11` 字节：

```text
[0xAA][idx][x1_L][x1_H][y1_L][y1_H][x2_L][x2_H][y2_L][y2_H][0xFF]
```

字段说明：

- `0xAA`：帧头
- `idx`：目标编号
- `x1, y1`：边框左上角
- `x2, y2`：边框右下角
- `0xFF`：帧尾

坐标字段均为小端 `uint16`。

### 8.3 `idx` 定义

- `idx = 0`：当前帧检测到目标
- `idx = 255`：当前帧无目标

本工程目前只输出单目标，因此有效目标统一使用 `idx = 0`。

### 8.4 逻辑坐标系

串口发送的不是原始摄像头 `320x240` 像素坐标，而是缩放后的逻辑坐标：

- `X: 0 ~ 159`
- `Y: 0 ~ 119`

也就是：

- `UART_VIEW_W = 160`
- `UART_VIEW_H = 120`

这样做的目的，是让下位机直接围绕固定中心点做控制：

- 目标中心 `center_x = 80`
- 目标中心 `center_y = 60`

### 8.5 有目标帧示例

```text
AA 00 32 00 28 00 58 00 4A 00 FF
```

解释：

- `idx = 0`
- `x1 = 50`
- `y1 = 40`
- `x2 = 88`
- `y2 = 74`

边框中心为：

```text
cx = (50 + 88) / 2 = 69
cy = (40 + 74) / 2 = 57
```

### 8.6 无目标帧示例

```text
AA FF 00 00 00 00 9F 00 77 00 FF
```

解释：

- `idx = 255`
- 坐标填充为整幅逻辑视图边界
- 这类帧必须解释为“无目标”
- 绝不能把它当成有效检测框继续参与 PD 控制

## 9. 与下位机联调

下位机目录：

`D:\2026_SmartCar\PersonalCode`

本次联调的关键约定如下。

### 9.1 解析规则

下位机必须：

- 按固定 `11` 字节帧解析
- 按小端格式解析 `x1 y1 x2 y2`
- 使用 `idx == 255` 判断无目标
- 使用边框中心 `(x1 + x2) / 2, (y1 + y2) / 2` 作为控制输入

### 9.2 控制目标

居中控制的目标点固定为：

- `center_x = 80`
- `center_y = 60`

误差定义推荐为：

```text
err_x = 80 - cx
err_y = 60 - cy
```

### 9.3 PD 控制建议

当前下位机建议使用纯 `PD`：

```text
output = kp * error + kd * d_error
```

建议：

- `ki = 0`
- 有目标时才更新 PD
- 明确收到 `idx = 255` 时立即停控并清空目标状态
- 只有短时丢包时，才允许保留上一帧结果做极短时间保持
- 对 `cx, cy` 建议做低通滤波，对电机占空比建议做爬坡限制

### 9.4 运动方向校准

如果视觉框是对的，但车运动方向不对，请优先检查：

- `vx` 正方向是不是对应小车右移
- `vy` 正方向是不是对应小车前进
- `err_x` 和 `err_y` 的符号是不是与底盘坐标一致
- 电机混控矩阵方向是否和实际装车一致

这一步不要靠猜，建议直接单独测试：

- 发送正 `vx`，观察是不是向右
- 发送负 `vx`，观察是不是向左
- 发送正 `vy`，观察是不是向前
- 发送负 `vy`，观察是不是向后

## 10. 参数调试建议

如果识别不稳，可以重点调整以下参数。

### 10.1 目标尺寸参数

文件：

`project/user/color_tracer.c`

关键参数：

- `COLOR_DEFAULT_WIDTH_MIN`
- `COLOR_DEFAULT_HEIGHT_MIN`
- `COLOR_DEFAULT_AREA_MIN`

目标太小识别不到时，可适当减小这些值。

### 10.2 形状约束参数

关键参数：

- `COLOR_DEFAULT_ASPECT_MIN`
- `COLOR_DEFAULT_ASPECT_MAX`
- `COLOR_DEFAULT_FILL_MIN`

如果背景里有同色块误检，可适当收紧这些范围。

### 10.3 网格步长

文件：

`project/user/color_tracer.h`

关键参数：

- `COLOR_GRID_STRIDE`

说明：

- 步长更小：检测更细，但计算量更大
- 步长更大：速度更快，但小目标更容易漏检

## 11. 常见问题

### 11.1 屏幕上有框，但下位机不动

先检查：

- 串口波特率是否为 `115200`
- `TX/RX` 是否交叉连接
- 是否共地
- 下位机是否确实收到 `0xAA ... 0xFF`

### 11.2 屏幕上有框，但运动方向相反

先检查：

- 下位机是否按 `160x120` 坐标系做控制
- `err_x`、`err_y` 的定义是否正确
- 电机方向和混控矩阵是否与实车一致

### 11.3 标定后还是经常丢目标

先检查：

- 光照是否变化太大
- 沙包颜色是否过暗或过亮
- 目标是否太小
- 标定区域是否混入背景

### 11.4 编译后表现像旧版本

处理方式：

1. `Rebuild All`
2. 重新下载
3. 手动复位
